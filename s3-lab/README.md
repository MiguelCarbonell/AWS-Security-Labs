# S3 Lab - Private and Versioned Bucket

## Objective
Create a secure S3 bucket to store internal files for a fictional company.

## What I did
- Created the bucket `miguel-empresa-archivos-2026`
- Uploaded a test file
- Kept public access blocked
- Enabled bucket versioning

## What I learned
- S3 is not an instance, it is storage
- A bucket can store files in a centralized way
- Blocking public access helps protect data
- Versioning allows you to keep changes and recover previous versions

## Risks avoided
- Accidental public exposure of files
- Loss of changes when overwriting a file
